FPGA 信号分离装置 —— FFT + 锁相环的混合架构

写在前面

这是 2023 年全国大学生电子设计竞赛的作品，题目是"信号分离装置"，拿了江苏省一等奖。核心任务：输入一个由两个不同频率正弦波叠加而成的混合信号，在 FPGA 里把两个信号分开，分别从两路 DA 独立输出。

听起来像是"把混在一起的鸡蛋和面粉再分开"——事实上也差不多。但 FPGA 做这件事有天然的劣势：不知道信号频率是多少。题目没说输入信号的具体频率，只给了一个范围。你得自己找出频率，然后锁定，然后跟踪，然后分离。

我们用了 FFT + 双通道锁相环 的混合架构：FFT 先扫一眼频谱，找到两根谱线的位置，算出对应的频率控制字，喂给两个 PLL 分别锁定。锁定之后，即使输入信号频率漂移，PLL 也能自动跟踪。

  • 平台: Xilinx Artix-7 xc7a100t + Vivado
  • 源码: fuben（即 separation 最终版）
  • 分工: 三人团队，硬件/FPGA/单片机各一人，我负责全部 FPGA 部分

竞赛题目要求

2023 年全国大学生电子设计竞赛 H 题——信号分离装置。核心要求：

双路输出信号源输出频率范围在 20kHz~100kHz 且 f_A < f_B、峰峰值均为 1V 的两路周期信号 A 和 B，经增益为 1 的加法器得混合信号 C，通过分离电路分离出信号 A’ 和 B’，要求 A’ 和 B’ 相比 A 和 B 波形无失真，且在示波器上能连续稳定同频显示。

各阶段要求递进：

  1. 基本要求：A=50kHz、B=100kHz 正弦波能正确分离；A 和 B 均为 10kHz 整数倍正弦波也能分离
  2. 发挥部分：A 和 B 分别为 5kHz 整数倍的正弦波或三角波时能分离
  3. 更高要求：A 和 B 均为正弦波且 f_B 是 f_A 的整数倍时，B’ 与 A’ 的初相位差可在 0°~180° 范围设置，分辨率 5°，误差 ≤5°

方案论证：为什么用锁相环而不是比较器？

题目最直观的思路是用滤波器——高通低通分开就行。但 f_A 和 f_B 可能只差 5kHz（比如 20kHz 和 25kHz），模拟滤波器的 Q 值根本达不到。数字滤波器也难——FIR 的过渡带在这个频段要做得很窄，阶数爆炸。

另一种思路是比较器整形 → 方波 → 再用什么办法还原。但从竞赛报告中的仿真图可以看到，比较器方案恢复出的波形"瑕疵、失真严重"——方波丢失了幅度和波形信息，还原不了三角波。

我们最终选择了FFT 谱分析 → NCO 信号重建 → 锁相环闭环锁定的路线。这个方案的核心理念是：不在时域硬分离，而是先看清频谱、再重新生成。

FFT 告诉你"混合信号里有什么频率分量"，NCO（DDS）根据这些频率重新生成干净的波形，PLL 确保生成的波形和原始信号相位同步。分离的本质不是滤波，而是"识别 → 重建 → 同步"。

系统架构

时钟分配：板载 50MHz → PLL → 10MHz（系统时钟）+ 5MHz（FFT 时钟）。

核心思路一句话：FFT 看全局，PLL 盯局部。 FFT 负责"找频率"，PLL 负责"跟着跑"。

核心模块详解

1. 数字锁相环 — 整个系统的灵魂

锁相环（PLL）是信号分离的执行者。它的原理和模拟锁相环完全一致，只是全部在数字域实现。

结构:

鉴相器：乘法器。输入信号 × 本地 NCO 的 sin 输出，乘积的直流分量正比于相位差。这是最朴素的鉴相方式，不需要 CORDIC，一个乘法器搞定。

环路滤波器：PI 结构。

这里有个很关键的设计：频率控制字 = FFT 前馈频率字 + 环路滤波器输出。FFT 给出的频率字是"粗调"，PLL 的反馈在"细调"。两者叠加，PLL 不需要从零开始捕获——FFT 已经告诉它目标大概在哪个位置，环路滤波器只需要把剩下的相位误差消掉就行。

这极大加速了锁定过程。纯 PLL 从自由振荡到锁定可能需要几百毫秒，有了 FFT 前馈，几十微秒内就能锁定。

DDS 相位累加器：

ROM 里存了一整周期的 cos/sin 波形（双口 ROM，一路出 cos 一路出 sin）和三角波。地址位宽 10bit = 1024 点，对于 10MHz 采样率下的几 MHz 信号来说精度足够。

2. FFT + 峰值检测 — 频率识别的眼睛

FFT 跑在 5MHz 时钟上，2048 点。xFFT IP 核配置为 forward FFT，输入位宽 10bit（匹配 ADC）。

3. 三角波/正弦波自动判断 — 谐波分析

题目要求能区分原始信号是正弦波还是三角波。这怎么自动判断？

核心洞察：三角波有丰富的奇次谐波（3次、5次、7次…），正弦波没有。 如果 FFT 发现在基频 3 倍位置有明显能量，那就是三角波。

阈值 peak1/16 是经验值——如果三次谐波幅度超过基波的 1/16，就认为这是三角波。在实际测试中，三角波的三次谐波通常在基波幅度的 1/9 左右，1/16 的阈值给了足够的余量。

重难点与踩坑记录

难点 1：FPGA 晶振与信号源晶振不一致——用频率锁相位

这是整个电赛过程中花时间最多的一个问题。

信号源输出的 A 和 B，和 FPGA 重建出的 A’ 和 B’，即使频率字完全一样，也会因为晶振的微小偏差导致频率不完全相等。这个偏差可能是几十 ppm（百万分之一），听起来没啥——但频率不相等意味着相位差在持续累积。一开始 A 和 A’ 可能是对齐的，几十秒后它们就错开了，屏幕上两个波形缓缓漂移。

正常思路是用 PLL 的相位误差来修正频率——鉴相器输出相位差，经环路滤波器调整 NCO 频率，把相位拉回来。这是标准 PLL 的做法。问题是：题目要求的是示波器上肉眼看到同频稳定，而不是锁到 p 秒级精度。 晶振偏差本身就在缓慢漂移，追相位的环路滤波器带宽太窄，追不上漂移速度。

最终的方案是：用频率来锁定相位。 每隔一段时间（约 0.5 秒），用 FFT 重新测一次频率，更新 PLL 的频率控制字。同时在两个 PLL 中任意一个的相位累加器上增加微调量，利用拍频振荡器的原理——两个频率微小不同的信号之间会产生缓慢变化的相位差——通过控制频率差来控制相位变化的速度和方向。

具体做法：

  1. 先用 FFT 精确测出 f_A 和 f_B（2048 点 FFT，分辨率 2.44kHz，足够定位到最近的频点）
  2. 用 LUT 映射得到频率控制字，设置两个 PLL
  3. 实时监测 A’ 和 B’ 的过零点，与期望相位比较
  4. 微调频率控制字来"追赶"相位偏差

精妙之处在于：不需要改变 FPGA 晶振，也不需要外部参考时钟，纯靠数字域的频率微调实现相位的长时稳定。

难点 2：PLL 捕获范围与锁定时间

纯 PLL 的捕获范围取决于环路滤波器的带宽。带宽越宽，捕获范围越大，但锁定后的抖动也越大；带宽越窄，锁定后越干净，但初始捕获慢甚至根本捕不到。

我们的解法：FFT 前馈直接把频率字预置到目标附近，PLL 只需要做小范围的相位微调。这样环路滤波器可以设得很窄，锁定后又稳又干净。

难点 2：两个频率太近时 FFT 分不开

FFT 的频率分辨率 = fs / N = 5MHz / 2048 ≈ 2.44kHz。如果两个输入信号频率差小于 2.44kHz，FFT 只看到一个峰。

实际测试中，题目要求的两个信号频率差至少在 10kHz 以上，所以 2048 点 FFT 够用。但如果需要更高分辨率，可以增大 FFT 点数（代价是更长的处理延迟和更多 BRAM）。

难点 3：FFT 输出延迟导致 PLL 启动滞后

FFT 从输入到输出有几十微秒的延迟（2048 点 × 200ns/点 ≈ 0.4ms）。在这段时间里 PLL 没有频率字，处于自由振荡状态。

解决方案：PLL 默认频率字设为一个中间值（约 1MHz），上电后先锁定到一个默认频率。FFT 第一帧结果出来后，频率字在下一个 FFT 周期开始时更新，PLL 快速重新锁定到新频率。

难点 4：CORDIC IP 截位导致幅度不准

Xilinx CORDIC IP 的幅度输出有个缩放因子（≈1.646），需要补偿。在峰值检测逻辑里我们用的是相对阈值（peak1/16），不需要绝对幅度准确，所以这个问题影响不大。

但在三角波/正弦波判断时，三次谐波的绝对幅度确实偏低。通过反复仿真调整阈值（从 peak1/8 调到 peak1/16），找到了一个对各种频率组合都适用的参数。

难点 5：版本管理混乱

fuben 目录下有两个版本的 FFT_Config（FFT_Config.v 和 FFT_Config_suanfa.v）、两个版本的顶层（mytop.v 和 PLL_top.v），testbench 连接的端口甚至和当前模块对不上。

这提醒我：电赛期间版本管理真的很重要。 如果现在重来，会用 Git 分支管理每次改动，而不是"另存为一个新文件"。

其中三角波判断这个功能，实测在信噪比较好的条件下几乎不出错。但如果加法器引入了失真（比如 LM358 在大信号下饱和），三次谐波可能被压缩，导致判断失败。这个问题和模拟电路部分有关，不是 FPGA 的问题。

写在最后

这个项目是我第一次把 FFT 和锁相环放在同一个系统里协同工作。做之前觉得"FFT 扫一下频谱不就完了吗"，做了之后才发现 FFT 只是告诉你频率在哪，真正把信号干净地分离出来，靠的是锁相环的连续跟踪。

电赛是团队赛，三个人分别负责硬件（加法器 + 模拟前端）、FPGA（我）、单片机（串口屏）。我负责全部 FPGA 部分，从方案设计到 Verilog 编码到上板调试一共四天三夜。最后验收的时候，评委说"你这个 FPGA 部分做得比很多研究生毕设都完整"——那一刻觉得熬夜都值了。

回顾这个项目，最让我受益的不是拿了省一，而是被逼着在短时间内把 FFT、锁相环、CORDIC、UART 这些模块串成一个可工作的系统。之前做频谱仪、做 DDS 频率计的时候，这些模块是独立学习的，各自理解也就各自忘了。但当它们必须在一个系统里互相配合时，你对每个模块在整个信号链中的角色才有了真正的认知——这大概就是电赛的意义。

如果你也在做类似的信号分离项目，强烈建议先理解锁相环的数学原理，再动手写代码。PLL 参数（环路滤波器系数、DDS 位宽、ROM 深度）之间的 trade-off 是真正体现功力的地方。
