CORDIC 算法 —— 从原理到 FPGA 实现

写在前面

CORDIC（Coordinate Rotation Digital Computer）是我在 FPGA 里用得最多的算法模块之一。它用移位和加减代替乘法器，能同时算 sin/cos、arctan、sqrt 等超越函数。从频谱仪的本振生成，到 AM/FM 解调的幅度相位提取，再到锁相环的鉴相器，CORDIC 几乎出现在我每一个 FPGA 信号处理项目里。

这篇教程从一个简单的几何直觉出发，一步步推导到完整的参数化 Verilog 实现。

一、核心直觉：旋转一个向量

假设平面上有一个向量 (x, y)，我想把它逆时针旋转角度 θ，得到新向量 (x’, y’)：

这个式子需要 cos θ 和 sin θ——但如果我们本来就不知道它们（比如要从 IQ 算相位），这就成了循环论证。

CORDIC 的巧思：不直接旋转 θ，而是把它拆成一系列固定的小角度旋转。 每次旋转的角度是 atan(2^(-i))，对应的 cos 和 sin 可以通过移位实现。

这个过程不需要任何乘法器。只有加法、减法、移位。

二、两种工作模式

ROTATE 模式：给定角度，求 sin/cos

目标：把 z 旋转到 0。

  • d_i = sign(z_i)：z_i > 0 就顺时针转（减角度），z_i < 0 就逆时针转（加角度）
  • 初始值：(x₀, y₀) = (K, 0)，其中 K = ∏ cos(atan(2^(-i))) ≈ 0.60725
  • 结果：(x, y) → (cos θ, sin θ)

这就是 DDS 替代方案——给定一个累加的相位（频率控制字不断累加），CORDIC 连续输出 cos 和 sin 波形。跟 ROM 查表法相比，省 BRAM，精度不受表深度限制。

VECTOR 模式：给定 (x, y)，求幅度和角度

目标：把 y 旋转到 0。

  • d_i = -sign(y_i)：y_i > 0 就顺时针转（让 y 减小），y_i < 0 就逆时针转（让 y 增大）
  • 初始值：(x₀, y₀) = (x_in, y_in)
  • 结果：x → K·sqrt(x² + y²)（幅度），z → arctan(y/x)（相位）

这正是 IQ 检波的核心——I/Q 两路信号输入 VECTOR 模式，x 输出 AM 包络，z 输出瞬时相位。相位差分就是 FM 频偏。

四、参数选择指南

参数 建议值 理由
ITERATIONS 12~16 12 次精度已够大多数应用，16 次达到 ~0.005° 相位精度
XY_BITS 12~16 配合 12~14 位 DAC/ADC
PH_BITS 32 频率分辨率 = f_clk / 2^32，100MHz 下约 0.023Hz
流水线延迟 ITERATIONS + 3 象限折叠和输出寄存器额外需要 3 拍

16 次迭代消耗约 16×(XY_BITS + 几个 FF) 个 LUT + FF 对。对于 12 位大约 300 个 LUT，跟一个 Xilinx CORDIC IP 核比多不了多少，但完全可控、可移植。

五、和 Xilinx CORDIC IP 的对比

维度 手写 CORDIC Xilinx CORDIC IP
灵活性 可任意加功能（相位累加器、象限折叠） 功能固定
可移植性 任何 FPGA 都能跑 锁定 Xilinx 生态
资源 略多（需要看综合结果） 高度优化
学习价值 深刻理解算法本质 黑箱

我个人倾向在理解原理后手写一版，后续项目直接复用。这个 Cordic.v 被我在频谱仪、锁相环、AM/FM 解调三个项目中用到了，每次只需改参数就行。

六、验证方法

最简单的验证：用 ROTATE 模式生成 cos/sin，画 Lissajous 图看是不是一个完美的圆。相位步进越小（频率控制字越小），圆越密实。

也可以写一个 testbench，把 ROTATE 输出的 cos/sin 直接喂给另一个 VECTOR 模式的 CORDIC，看相位输出是否等于原始相位输入。如果闭环验证通过，说明两个模式都是对的。

写在最后

CORDIC 是那种"知道之后就觉得理所应当"的算法。它把一个需要乘法器和 ROM 表的任务，退化成了只需要加法和移位的流水线。在 FPGA 上，加法器和移位器遍地都是，乘法器和 BRAM 才是稀缺资源——CORDIC 完美匹配了这种硬件特性。

如果你想深入理解，推荐 Jack Volder 1959 年的原始论文，以及 Xilinx 的 CORDIC IP 文档（里面有很好的参数选择说明）。
