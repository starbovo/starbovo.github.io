用FPGA自制扫频式频谱仪（六）：数字检波、视频滤波与整机联调

发表于2023-07-13|更新于2026-04-29|FPGA
|总字数:1.7k|阅读时长:5分钟|浏览量:

前情提要

经过前面五篇，我们已经完成了扫频信号生成、射频信号模拟、IQ 混频、CIC 多级抽取、RBW 低通滤波。I/Q 两路基带信号已经干干净净地躺在那里了。

但那还是 I 和 Q 两路——不是频谱幅度。这一篇要把它们变成最终的频谱值，并上板跑起来。

数字检波：从 IQ 到幅度

IQ 基带信号是一对正交分量，幅度信息藏在 sqrt(I² + Q²) 里。这块有两种常见做法：

方案一：CORDIC 矢量模式。 把 I/Q 输入 CORDIC 的 VECTOR 模式，直接旋转变换成幅度和相位。优点是精度高，缺点是 CORDIC 有流水线延迟。

方案二：直接平方和。 amplitude = I² + Q²，不开根号。频谱仪显示的是相对幅度，开不开根号只影响纵坐标的刻度形状，不影响峰值位置的判断。

我们选了方案二——省资源，省延迟，够用。

这里有个小细节：用组合逻辑直接算 a*a + b*b，没有流水线。这在 50kHz 的低采样率下没问题，但如果你跑在 100MHz 的时钟域，一定要加寄存器切流水，不然时序必炸。

检波后的输出波形：

可以看到包络被成功提取，但波形上还有些毛刺和纹波。这就是接下来要解决的。

视频滤波（VBW）：给频谱"磨皮"

检波输出的包络上还残留着高频纹波——这是混频产物和滤波器过渡带的残余。视频滤波器（Video Bandwidth, VBW）就是一个低通滤波器，把这些纹波滤掉，让频谱显示更平滑。

本质上 VBW = 对已检波信号做时间上的平均。VBW 越小，平均时间越长，频谱越平滑，但对瞬态信号的响应越慢。

我们设计了三个 VBW 档位：

VBW 档位 截止频率 效果
50kHz 最高 基本不过滤，能看到快速变化的信号
5kHz 中等 适中平滑
1kHz 最低 最强平滑，噪声基底最干净

实现和 RBW 滤波器一样：MATLAB FDATool 设计低通滤波器 → 导出 COE → Vivado FIR IP。只不过这次的截止频率宽松得多，滤波器阶数也很小。

增大滤波器过渡带宽度后，没有 VBW 的样子：

像是素颜照片，能看到所有细节但不够干净。加上 VBW 之后就像开了磨皮，噪声被压下去，谱线更清晰。

横纵坐标：频谱图的 XY 轴怎么来

这个问题很关键但经常被忽略。

纵轴（幅度）：就是检波+VBW后的幅度值。通过串口/并口送给单片机或直接接 DA 输出。

横轴（频率）：不需要额外计算。扫频本振的频率控制字本身连续变化，它和横坐标是一一对应的。如果你把 DA 输出的幅度接到示波器的 Y 通道，把扫频控制字接到 X 通道，示波器 XY 模式就能直接显示频谱。

横轴的频率映射关系：

扫频范围 = [center_freq - SPAN/2, center_freq + SPAN/2]

整机联调：从仿真到上板

顶层模块 top_on_board.v 是整个系统的板级入口。来看看它的结构：

几个板级适配细节：

  1. ADC 数据格式：大多数高速 ADC 输出偏移二进制码（Offset Binary），而 FPGA 内部处理用补码（Two’s Complement）。直接把 MSB 取反就完成了转换。

  2. 时钟：板载 50MHz 晶振进 PLL，产生 10MHz 和 100MHz 两路时钟分别给不同模块。da_clk 取反是为了让 DAC 在时钟的另一沿采样，给数据留足建立时间。

  3. 按键消抖：key_filter.v 实现 20ms 消抖。原理很简单——计数器计数到 20ms 阈值才认为按键有效。

已知问题和改进方向

  1. 滤波器延迟导致频率轴偏移：FIR 滤波器的群延迟会让输出幅度落后于扫频控制字，导致频谱横坐标不准。最简单的补偿是把扫频控制字也延迟同样的拍数。

  2. CIC 通带衰减没有补偿：多级 CIC 的通带有 sinc 函数 roll-off，在扫频范围边缘幅度会偏低。加一级 CIC 补偿 FIR 就能修正。

  3. DAC 输出截位简单粗暴：SquareSum 输出 32 位，直接取高 10 位给 DAC。实际信号动态范围可能很大，加对数转换会更像真实频谱仪的 dBm 显示。

  4. 没有做对数刻度：真实频谱仪纵轴是 dB，我们输出的是线性幅度。如果后面接带串口屏的单片机，可以在 MCU 端做对数转换。

写在最后

六篇文章写完，这个频谱仪教程系列总算完整了。从扫频原理到中频选择，从 CORDIC 到 CIC 抽取，从 MATLAB 滤波器设计到 Vivado IP 配置，从 IQ 检波到上板联调——我把踩过的坑和想通的点都写在这里了。

这个项目最让我受益的，不是最终跑出频谱图的成就感，而是在设计过程中被迫理解了多速率信号处理和软件无线电的基本思想。这些概念在后来做锁相环和信号分离的时候反复用到，构成了我对 FPGA 数字信号处理的基本认知框架。

如果你也在做类似的项目，希望这几篇能帮你少走一些弯路。有问题欢迎在评论区讨论。
