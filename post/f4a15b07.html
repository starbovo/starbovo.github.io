数字系统综合设计 - DDS 信号发生器与测频仪

发表于2022-11-06|更新于2026-04-29|FPGA

数字系统综合设计 - DDS 信号发生器 + 测频仪 + 蓝牙控制

项目概述

基于 EGO1 FPGA 开发板（Xilinx Artix-7）和 Vivado 的综合数字系统设计。使用 Verilog HDL 实现了 DDS 正弦波发生器、频率测量仪、蓝牙串口遥控、呼吸灯、AM 调幅等功能的集成系统。

  • 作者: 赵婧萱（starbovo）
  • 平台: EGO1（100MHz 主频）+ Vivado
  • 源码: GitHub - Digital-System-Design2022

创新与难点

1. 双门限迟滞比较器 — 抗噪声正弦波转方波

问题: 测频模块需要将 AD 采样的 10bit 正弦信号转为方波。简单阈值比较（"一刀切"法）在有噪声毛刺时会产生大量误翻转，精度严重下降。实测峰峰值 1V 时几乎不可用。

创新方案: 引入迟滞比较器思想，设置上下双门限，配合状态机消抖。

MATLAB 仿真对比证实：双门限法在含噪信号下能输出干净的方波，而一刀切法产生大量毛刺。

2. DDS 双精度架构 — 根据精度需求选型

难点: 开关控制 DDS 和蓝牙控制 DDS 的精度要求完全不同：

DDS 相位累加器 ROM 深度 ROM 位宽 频率步进 适用场景
DDS8（开关） 8bit 256 8bit ~39kHz 步长要求 < 40kHz
DDS14（蓝牙） 14bit 16384 14bit ~610Hz 误差要求 < 2kHz

核心原因：蓝牙要求误差 < 2kHz，意味着频率步进至少 < 4kHz。4kHz / 10MHz = 2^14，所以相位累加器至少 14bit。

频率控制字计算:

  • 开关模式: fout = K × 10MHz / 2^8，K ∈ [0, 51]（上限 2MHz）
  • 蓝牙模式: fout = K × 10MHz / 2^14，K 由手机输入的十进制频率转换而来

3. 资源节约型 BCD↔二进制转换

难点: 用除法/取模做进制转换会消耗大量 LUT 资源且不可综合。

创新:

  • 二进制→BCD: 采用"加 3 左移法"（Shift-Add-3），仅需移位和加法
  • BCD→二进制: 串行迭代 data_bin = data_bin × 10 + digit，其中 ×10 用移位加法 ×8 + ×2 实现，不需乘法器

4. 溢出保护 + 模式切换无缝集成

所有模块合在一个顶层运行是验收硬性要求。顶层通过 sw_mode[1:0] 切换四种显示模式（学号/开关频率/测频结果/蓝牙频率），通过 dds_dip 切换 DDS 信号源。

系统架构

各模块详解

时钟分频 (freq_div10.v)

参数化偶数分频器，默认 100MHz → 10MHz（DIV_0CLK=10）。计数到 N/2-1 翻转时钟，占空比 50%。

直接测频法 (cymometer_direct.v)

  • 固定闸门时间 0.5s，每 1s 更新一次测量结果
  • fx = cnt × fsys / N（cnt=闸门内脉冲计数，N=闸门时钟计数）
  • 误差 = 1/cnt × 100%，信号频率越高误差越小

实测结果:

条件 理论频率 实测频率 误差
K=1 测频 39.062kHz 39.062kHz 0
K=46 测频 1.796875MHz 1.796860MHz -0.00083%
蓝牙设 780kHz 780kHz 781.25kHz 1.25kHz(<2kHz)

蓝牙串口 (bt_module/)

  • 波特率 9600bps，8 数据位，无校验
  • 手机输入 7 位十进制频率 → UART RX → 命令解析 → 28bit BCD
  • 经 bcd_bin 转二进制 → 溢出检查 → 计算 12bit 频率控制字 → DDS14 输出
  • 蓝牙模块各子模块（波特率发生器、UART RX、命令解析、消抖、同步器）均独立设计

数码管动态扫描 (seg_display.v)

共阴极 8 位数码管，分为左右两组。片选信号 an[3:0] 以约 76Hz 轮流选通，利用视觉暂留实现稳定显示。

PWM 呼吸灯 (pwm.v)

每 1ms PWM 周期内，占空比从 0→100%（100 步递增），再从 100%→0（递减），完整呼吸周期约 2 秒。

AM 调制 (AM_create.v)

cos_make.v 产生 10kHz 调制信号和 1MHz 载波信号（双口 ROM + 相位累加器），叠加直流偏置后经 MULT IP 核相乘输出。

注意事项

  1. 编码问题: Verilog 源文件使用 GB2312 编码（Vivado 默认），用 VSCode 等编辑器打开需切换编码，否则中文注释乱码
  2. 复位信号只能使用一次: 验收硬性要求，上电复位后不能再按复位键
  3. 所有功能合在顶层模块中运行: 不能单独下载各模块 bitstream，必须集成验证
  4. 第一遍验收即定成绩: 没有第二次机会，验收前充分测试
  5. ROM 深度选择: 频率精度要求越高，ROM 深度越大，但消耗 BRAM 资源也越多
  6. 双门限阈值: 代码中取 AD_in[9:5]（高 5 位）与 zero[9:5] 比较，等效于 32 级量化，适合 EGO1 板载 AD 的动态范围

调试工具

  • XCOM V2.6 — 串口调试助手，验证蓝牙数据收发
  • 波形数据生成器 — 生成测试波形
  • 蓝牙 app (base.apk) — 手机端发送频率数据
